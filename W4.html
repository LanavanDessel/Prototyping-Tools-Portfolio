Webpagina’s maken met HTML en CSS

Lana van Dessel

Datum: 16 oktober 2025

Tijdens de vierde les van Prototyping Tools leerden we hoe we onze ontwerpen kunnen omzetten in echte webpagina’s. De focus lag op HTML en CSS — de twee basistalen waarmee websites worden opgebouwd en vormgegeven. Het doel was om een digitale versie van ons logboek te maken, waarbij we alle lessen en reflecties overzichtelijk konden presenteren. Deze les ging dus niet alleen over code, maar ook over structuur, overzicht en de manier waarop inhoud en vorm samenwerken.

Wat ik heb gedaan

We begonnen met een duidelijke uitleg over de opbouw van een HTML-document: de head, body en verschillende elementen zoals <header>, <main> en <footer>. Daarna gingen we stap voor stap aan de slag met het maken van onze eigen webpagina. Ik koos ervoor om mijn logboek uit eerdere lessen als basis te nemen en dit om te zetten naar een echte website met meerdere pagina’s.

Vervolgens leerden we hoe je CSS kunt gebruiken om de website visueel aantrekkelijk te maken. Ik experimenteerde met lettertypes, kleuren en marges om alles beter op elkaar af te stemmen. Ook probeerde ik om de navigatie duidelijk te maken met knoppen die er netjes en modern uitzagen. In het begin was het lastig om de juiste opmaak te vinden, vooral omdat kleine fouten in de code meteen invloed hebben op de hele lay-out. Toch begon ik steeds beter te begrijpen hoe HTML en CSS samenwerken.

Daarnaast oefende ik met het gebruik van flexbox om kolommen en secties goed uit te lijnen. Dat was nieuw voor mij, maar ik merkte al snel hoe handig het is om een website flexibel te maken, zodat hij ook goed werkt op verschillende schermgroottes. Aan het einde van de les had ik een eerste versie van mijn logboekwebsite klaar. Het zag er nog niet perfect uit, maar het voelde goed om te zien dat mijn ontwerp uit Figma nu echt begon te “leven” op het scherm.

Reflectie

Deze les vond ik erg interessant, maar ook uitdagend. In het begin voelde HTML en CSS best technisch aan, vooral omdat ik gewend was om te werken met visuele tools zoals Figma. Toch merkte ik dat het werken met code me meer inzicht gaf in hoe websites echt zijn opgebouwd. Het gaf me een gevoel van controle en creativiteit tegelijk — alsof ik niet alleen iets ontwerp, maar het ook echt tot leven breng.

Wat ik nog lastig vond, is het behouden van overzicht in mijn code. Soms raakte ik de draad kwijt door alle tags, haakjes en klassennamen. Daarom heb ik geleerd om mijn HTML netjes te structureren met inspringingen en duidelijke namen voor onderdelen. Ook heb ik beter begrepen hoe belangrijk het is om consequent te werken met stijlen in CSS, zodat de website overzichtelijk en professioneel blijft.

Voor de volgende lessen wil ik me verder verdiepen in CSS-layouts, zoals grid en geavanceerde animaties. Daarnaast wil ik leren hoe ik mijn website interactiever kan maken — bijvoorbeeld door knoppen die bewegen of kleur veranderen. Mijn doel is om een website te maken die niet alleen informatief is, maar ook prettig aanvoelt voor de gebruiker. Deze les heeft me echt gemotiveerd om meer te leren over webdevelopment en om te zien hoe design en techniek samenkomen.
